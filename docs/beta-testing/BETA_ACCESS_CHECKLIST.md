# Beta Access Checklist

Date: 2026-07-01

Status: Checklist only. Do not implement or mutate access from this document.

## Tester Approval

- [ ] Tester approved by product owner.
- [ ] Tester role documented.
- [ ] Tester belongs to allowed first cohort.
- [ ] Tester is not a paying customer or public signup user.
- [ ] Tester does not require production compliance, live bank feeds, real payment collection, production email delivery, or sensitive production data entry.

## Acknowledgement

- [ ] Acknowledgement completed.
- [ ] Restrictions sent.
- [ ] Tester understands controlled beta only.
- [ ] Tester understands no official accounting, legal, tax, filing, audit, or compliance reliance.

## Account and Role

- [ ] Account provisioned.
- [ ] Least-privilege role assigned.
- [ ] Owner/Admin access kept internal unless separately approved.
- [ ] Test organization assigned.
- [ ] Demo/test data confirmed.
- [ ] No shared password recorded in docs.
- [ ] Credentials/invite sent only through approved private channel.

## Tester Materials

- [ ] Onboarding guide sent.
- [ ] Walkthrough script sent.
- [ ] Feedback form sent.
- [ ] Support contact sent.
- [ ] Escalation rules sent.

## Schedule

- [ ] Start date recorded.
- [ ] End date recorded.
- [ ] Access revoke date recorded.
- [ ] Expected walkthrough script recorded.
- [ ] Support owner recorded.
- [ ] Triage owner recorded.

## Safety Review Before Invite

- [ ] No production data will be used.
- [ ] No live provider, ZATCA, UAE, Peppol, ASP, bank, payment, email, object-storage, signed-URL, migration, seed/reset/delete, backup/restore, or destructive action is part of the script.
- [ ] Revocation checklist is ready.
- [ ] Owner approval recorded.
