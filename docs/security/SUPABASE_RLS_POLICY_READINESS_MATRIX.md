# Supabase RLS Policy Readiness Matrix

Status: draft readiness matrix only, not executed
Date: 2026-07-03

This matrix is generated from `docs/security/evidence/TENANT_RELATIONSHIP_GRAPH.json` and is a planning artifact only. It does not enable RLS, create policies, change roles, run SQL, run migrations, inspect hosted grants, or claim database-enforced tenant isolation is active.

## Safety Contract

- No executable SQL policies are included.
- No Supabase or database connection is required.
- App data remains API-mediated until a separate approved RLS/runtime-role goal runs in disposable non-production infrastructure.
- Data API exposure must remain blocked or explicitly reviewed before any browser role receives table access.

## Status Legend

- `ready-for-policy-design`: direct tenant key is present and can seed draft RLS policy design.
- `needs-parent-path-review`: policy design must prove the parent join path and write checks.
- `global-reference`: shared reference/configuration data; exposure and write ownership need separate grants review.
- `system/admin-only`: identity, root, audit, evidence, or admin-only data; prefer API/admin-only paths.
- `blocked`: no safe tenant path is known from static analysis.
