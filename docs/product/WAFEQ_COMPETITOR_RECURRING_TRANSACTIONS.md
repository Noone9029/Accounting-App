# Wafeq-Competitor Recurring Transactions

This document describes LedgerByte product positioning, not a current Wafeq feature assessment.

LedgerByte’s differentiator is accountant review before posting: one dense queue covers recurring invoices, bills, expense proposals, and journals; each attempt shows template version, timezone, trigger, blocker, retry count, and draft target. Templates preserve dimensions and explicit FX evidence without silent accounting.

The finance-first web workspace provides filters, attention counts, compact tables, lifecycle actions, Run Now, generated-target links, and one editor for schedule, document fields, currency, cost centers, and projects. Tables scroll inside panels; identifiers and dates are direction-contained for RTL layouts. Mobile users can review the same evidence without a native app.

Migration tooling offers CSV preview, row errors, duplicate and reference validation, reviewed serializable commit, and safe exports. Imports never activate templates automatically.

This is burner/demo user testing only. No provider or compliance claim is made: there is no live bank feed, payment collection, email delivery, OCR, external storage, ZATCA/UAE filing, or money movement. The next recommended product arc is the accountant month-end close workspace, consuming recurring readiness without expanding this engine’s boundary.
