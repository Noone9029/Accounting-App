# Support Incident Simulation Playbook

Status: controlled-beta support drill templates
Date: 2026-07-02

These simulations are for local/read-only support readiness. They do not run production drills, mutate customer data, call providers, send email, execute storage operations, or claim live monitoring.

| Simulation | Severity | Support checks first | Do not do | Escalation | Evidence to capture | Stop-the-beta condition | Response template |
| --- | --- | --- | --- | --- | --- | --- | --- |
| API down | SEV-1 if total outage; SEV-2 if degraded | `/health`, `/readiness`, deployment SHA, recent release notes, affected routes | Do not restart or redeploy without owner approval | Engineering owner, support owner | Timestamp, environment URL, status code, route, screenshot, SHA | Core tester workflows cannot continue | "We are investigating an API availability issue in the beta environment. Please pause testing on affected workflows until we confirm recovery." |
| Web down | SEV-1 if login/app shell unavailable | Web URL, login route, dashboard route, Vercel deployment status if accessible by approved operator | Do not mutate Vercel settings or env vars | Engineering owner | URL, browser, screenshot, SHA if visible | Login or dashboard unavailable for beta testers | "We are investigating web availability for the beta environment and will update when the route is usable again." |
| Database readiness degraded | SEV-1 if data integrity risk; SEV-2 otherwise | API readiness database status, affected workflows, recent DB-related changes | Do not run migrations, seed, reset, delete, or manual SQL | Engineering owner, data owner | Readiness output, error class, affected route, timeframe | Database status is unavailable or data integrity is uncertain | "Database readiness is degraded. Please stop data-entry testing until we confirm the beta data path is safe." |
| Email outbox stuck | SEV-3 unless access/security impacted | Email readiness, outbox status, retry plan, monitoring evidence | Do not send test email or force retries without approval | Support owner, engineering owner | Outbox counts/status only, no message bodies | Invite/password-reset testing depends on stuck delivery | "Email delivery appears delayed in beta. We are checking the outbox state and will avoid resending until the cause is confirmed." |
| Backup evidence stale | SEV-2 for beta expansion; SEV-3 otherwise | Backup readiness, restore evidence index, evidence age | Do not run restore or destructive backup commands | Data owner | Evidence file path, age, last verified status | Evidence missing before expansion or high-risk testing | "Backup/restore evidence needs review before expanding beta scope. Current testing may continue only if no data integrity issue is present." |
| Object storage blocked | SEV-3 unless documents inaccessible | Storage readiness, object-storage proof evidence, document route status | Do not upload/download private bodies or generate signed URLs | Storage owner | Readiness state, affected document metadata only | Document workflow requires unproven storage behavior | "Object storage remains blocked for beta readiness. We will use only approved document metadata and avoid private file operations." |
| ZATCA disabled/no-network confusion | SEV-2; SEV-1 if network attempted | ZATCA readiness/no-network evidence, user-facing copy, affected route | Do not run ZATCA commands or claim compliance | Compliance owner | Screenshot/copy, route, command not run confirmation | Any unexpected ZATCA network call or production-compliance claim | "ZATCA production behavior is not enabled in this beta. Please treat the screen as readiness/status information only." |
| UAE ASP unavailable/not connected | SEV-2; SEV-1 if real ASP call attempted | Compliance ASP readiness, provider status, no-network docs | Do not test provider credentials or call ASP endpoints | Compliance owner, security owner | Route, screenshot, readiness state, provider-call absence | Any real ASP network call before approval | "UAE ASP access is not connected for this beta. We are tracking readiness only and will not send data to an ASP." |
| Wrong balance reported | SEV-1 if data integrity likely; SEV-2 otherwise | Report/source transaction route, organization, timestamp, expected vs actual, recent actions | Do not edit postings, journals, VAT, inventory valuation, or reconciliation state | Accounting owner, engineering owner | Redacted screenshot, route, IDs, reproduction steps | Balance mismatch affects core beta trust or may indicate cross-tenant issue | "We are treating the reported balance as a data-integrity triage item. Please avoid further edits to that record until reviewed." |
| Missing invoice/PDF | SEV-2 if core workflow blocked; SEV-3 otherwise | Invoice route, generated-document status, PDF route, storage readiness | Do not regenerate/delete/archive documents without approval | Support owner, document owner | Invoice ID, document metadata, route, screenshot, no document body | Customer-visible document workflow blocked | "We are checking the document metadata and PDF workflow. Please do not regenerate the invoice document until we confirm the safe path." |
| Bank reconciliation mismatch | SEV-1 if posted ledger impact; SEV-2 otherwise | Reconciliation report, statement transaction, matching state, recent actions | Do not auto-match, post, reopen, or close reconciliation | Accounting owner | Redacted report data, route, item IDs, expected vs actual | Reconciliation close/approval cannot be trusted | "We are reviewing the reconciliation mismatch. Please pause reconciliation changes for this account until the review is complete." |
| Permission/access issue | SEV-1 for cross-tenant exposure; SEV-2 for blocked role | User role, organization, route, visible action, expected permissions | Do not grant broad permissions as a workaround | Security owner, support owner | User email, organization, route, screenshot, role, no tokens/passwords | Unauthorized data visible or revocation fails | "We are reviewing an access/permission issue. Please do not share screenshots containing private data until we confirm the safe evidence path." |
| Suspicious data leakage | SEV-1 | Organization boundary, route/API, screenshot metadata, affected tester | Do not copy leaked data, delete evidence, or continue affected testing | Security owner, data owner, support owner | Sanitized metadata only, route, time, user/org, exact access path | Any confirmed cross-tenant or secret/customer-data exposure | "Please stop using the affected beta route. We are treating this as a security incident and will preserve only sanitized evidence." |

## Drill Cadence

- Run tabletop review before adding a new beta cohort.
- Re-run after any SEV-1/SEV-2 incident.
- Keep owner placeholders until real support ownership is approved.
